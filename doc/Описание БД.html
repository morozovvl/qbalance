Таблица "счета"


Хранит бухгалтерский план счетов, т.е. список всех счетов, с которыми работает программа, в том числе забалансовых. Каждому счету можно сопоставить справочник. В таком случае счет становится аналитическим. В противном случае счет синтетический. В этой же таблице указывается, является ли счет балансовым или забалансовым и ведется ли на этом счете количественный учет.


"код" serial NOT NULL

Уникальный идентификатор счета

"имя" character varying(200) DEFAULT ''::character varying

Наименование счета, как это принято в плане счетов

"счет" character(5) DEFAULT ''::bpchar

Номер счета. Состоит из 5 символов. Например: "40", "51", "10.3", "41/1" и т.п.

"имясправочника" character varying(50) DEFAULT ''::character varying

Наименование справочника, который хранит объекты учета для аналитического счета

"баланс" boolean DEFAULT true

Флаг, который обозначает, является ли счет балансовым или забалансовым

"количество" boolean DEFAULT false

Флаг, который обозначает, ведется ли на счете количественный учет



Таблица "топер"


Хранит описание бухгалтерской операции. Под операцией подразумевается набор проводок, соответствующих одной строке табличной части документа. Например, при приобретении товара по безналичному расчету, в случае учета по покупной стоимости, для каждой приобретенной позиции делается проводка 41/60 (с кредита 60 счета в дебет 41 счета). В данном случае операция состоит из одной проводки. В случае учета по продажным ценам для каждой позиции делается две проводки 41/60 и 41/42. В этом случае операция состоит из двух проводок.

Так же таблица хранит информацию, используемую при формировании внешнего вида формы документа.


"код" serial NOT NULL

Уникальный идентификатор проводки

"опер" integer

Номер бухгалтерской операции

"номер" integer

Номер проводки в пределах операции

"дбсчет" character varying(5) DEFAULT ''::character varying

Дебетовый счет

"крсчет" character varying(5) DEFAULT ''::character varying

Кредитовый счет

"имя" character varying(100) DEFAULT ''::character varying

Наименование операции. Особое значение имеет для первой проводки в операции, т. к. эта информация отражается в меню операций на стороне клиента.

"итоги" character varying(1) DEFAULT ''::character varying

Содержит знак «+», если сумму проводки нужно прибавлять к итоговой сумме документа, или «-», если нужно вычитать. В остальных случаях сумма проводки не отражается на сумме документа.

"меню" boolean DEFAULT false

Содержит True, если операция отображается в списке операций на стороне клиента. Имеет смысл для первой проводки в операции

"осндокумент" character varying(20) DEFAULT ''::character varying


"нумератор" character varying(40) DEFAULT ''::character varying


"однаоперация" boolean

Содержит True, если документ может содержать только одну строку в табличной части. Например, платежное поручение

"кол" boolean DEFAULT false


"дбпост" boolean DEFAULT false

Содержит True, если дебетовый счет во всем документе постоянный для данной проводки. В этом случае справочник дебетового счета помещается в шапку формы. Например, при реализации товара за безналичный расчет (проводка 60/41), контрагент в пределах документа постоянный. В этом случае поле должно содержать True

"крпост" boolean DEFAULT false

Содержит True, если кредитовый счет во всем документе постоянный для данной проводки.

"дбдобав" boolean DEFAULT false


"крдобав" boolean DEFAULT false


"дбвыбор" boolean DEFAULT false


"крвыбор" boolean DEFAULT false


"дбсалвидим" boolean DEFAULT true


"крсалвидим" boolean DEFAULT true

Таблица "сальдо"


Хранит количественные и суммовые обороты, а так же остатки и сальдо для каждого объекта учета на каждом бухгалтерском счете. Бухгалтерский счет задается последовательностью из 5 символов и соответствует списку счетов (плану счетов) в таблице счета. Дебетовые и кредитовые обороты хранят значения оборотов для данного объекта учета за весь период существования БД и предназначены для быстрого вычисления текущих остатков и текущего сальдо. Если нужно получить значения дебетовых и кредитовых оборотов за конкретный период, то нужно воспользоваться соответствующей хранимой процедурой.


"счет" character(5) NOT NULL

Бухгалтерский счет объекта учета. Список счетов указан в таблице счета

"код" integer NOT NULL

Код объекта учета. Действителен в пределах справочника, который действует для указанного в поле счет бухгалтерского счета. Имя такого справочника можно прочитать в поле имясправочника в таблице счета.

"кол" numeric(10,3) DEFAULT 0

ВОЗМОЖНО БУДЕТ УДАЛЕНО ИЗ БАЗЫ!!!

"сальдо" numeric(14,2) DEFAULT 0

ВОЗМОЖНО БУДЕТ УДАЛЕНО ИЗ БАЗЫ!!!

"дбкол" numeric(10,3) DEFAULT 0

Дебетовый количественный оборот за все время существования БД

"дебет" numeric(14,2) DEFAULT 0

Дебетовый оборот за все время существования БД

"кркол" numeric(10,3) DEFAULT 0

Кредитовый количественный оборот за все время существования БД

"кредит" numeric(14,2) DEFAULT 0

Кредитовый оборот за все время существования БД

"конкол" numeric(10,3) DEFAULT 0

Остаток на текущий момент

"консальдо" numeric(14,2) DEFAULT 0

Сальдо на текущий момент

"концена" numeric(14,2) DEFAULT 0

Цена на текущий момент


Имеет триггера:

считатьконечноесальдо — вычисляет конечный остаток и сальдо при изменении оборотов для объекта учета. Вызов, как правило, происходит при добавлении, изменении, удалении проводки, в которой упомянут соответствующий объект учета.

testdeleting_сальдо — триггер вызывается при попытке удалении записи. Проверяет, имеются ли в БД проводки, которые ссылаются на эту запись. Если таковые имеются, то запись не удаляется, а пользователю отправляется соответствующее уведомление.

"код" serial NOT NULL

Уникальный идентификатор проводки

"дбсчет" character(5) NOT NULL

Дебетовый счет проводки

"дбкод" integer NOT NULL

Код объекта учета на дебетовом счете

"крсчет" character(5) NOT NULL

Кредитовый счет проводки

"кркод" integer NOT NULL

Код объекта учета на кредитовом счете

"кол" numeric(10,3) DEFAULT 0

Количество

"цена" numeric(14,2) DEFAULT 0

Цена

"сумма" numeric(14,2) DEFAULT 0

Сумма

"стр" integer NOT NULL

Номер строки в документе

"доккод" integer NOT NULL

Ссылка на код документа в таблице документы

"опер" numeric(3,0) NOT NULL

Номер типовой бухгалтерской операции. Номера возможных операций хранятся в поле опер таблицы топер

"номеропер" numeric(1,0) NOT NULL

Номер проводки в пределах бухгалтерской операции


Имеет триггера:

вставитьпроводку — срабатывает при добавлении новой проводки. Если поле топер.считать для этой проводки равно True, то есть проводку такого типа нужно проводить, то триггер добавляет записи в таблицу сальдо для комбинации дбсчет+дбкод и крсчет+кркод при условии, что счета дбсчет или крсчет не являются забалансовыми. Если записи дбсчет+дбкод или крсчет+кркод в таблице сальдо существуют, то они не добавляются. Если счет, который указан в поле дбсчет или кркод по какой-либо причине отсутствует в таблице счета, то он в нее добавляется. После этого происходит вычисление остатков и сальдо для заданных комбинаций дбсчет+дбкод и крсчет+кркод в два этапа. На первом этапе вычисляются обороты в таблице сальдо для каждой комбинации дбсчет+дбкод и крсчет+кркод. Оборот по количеству по дебету вычисляется как сальдо.дбкол = сальдо.дбкол + проводки.кол. Суммовой оборот по дебету вычисляется как сальдо.дебет = сальдо.дебет + проводки.сумма. Аналогично происходят вычисления кредитовых оборотов. На втором этапе срабатывает триггер считатьконечноесальдо в таблице сальдо, который вычисляет окончательное сальдо для указанных комбинаций. Остаток вычисляется как сумма полей сальдо.конкол = сальдо.кол + сальдо.дбкол — сальдо.кркол. Аналогично вычисляется сальдо: сальдо.сальдо = сальдо. сальдо+ сальдо.дебет — сальдо.кредит.


считатьпроводку — срабатывает при изменении существующей проводки. Вычисления остатков и сальдо на соответствующих счетах происходит аналогично триггеру вставитьпроводку. Разница с ним заключается в том, что этот триггер обрабатывает и те случаи, когда изменилось значение полей дбсчет, дбкод, крсчет или кркод.


удалитьпроводку — срабатывает при удалении проводки. Вычисления остатков и сальдо на соответствующих счетах происходит аналогично триггеру вставитьпроводку. Записи в таблице сальдо не удаляются.
